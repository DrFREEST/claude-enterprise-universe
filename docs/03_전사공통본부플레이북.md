# 전사 공통본부(Shared Services) 플레이북 (Enterprise 기준)
- 작성일: 2026-02-12

---

## 운영본부(변경관리/CAB, 릴리즈, SRE)
- 입력: 사업부 변경요청(CR), 품질/보안 게이트 결과, 운영 리스크
- 출력: 변경 승인/반려, 릴리즈 캘린더, 롤백 플랜, 인시던트 대응
- KPI: 변경 실패율, 릴리즈 정시율, MTTR, 가용성(SLO)
- 권한: **L2**

---

## 플랫폼본부(파이프라인/런타임/관측/비용)
- 입력: 사업부 요구, 운영 정책, 보안 정책
- 출력: 표준 파이프라인, 런타임 격리/스케줄링, 관측 스택, 비용 통제 도구
- KPI: 배포 리드타임, 실행 성공률, p95 지연, 단가 절감
- 권한: L0~L2(도구 제공; 최종 배포 승인권은 운영본부)

---

## 보안·정책·준법본부
- 입력: 위협모델, 감사 요구, 고객 보안요구서, 사고/취약점 데이터
- 출력: Policy-as-Code, 보안 스캔/비밀관리, 준법 기준/감사 대응
- KPI: 정책 위반 0건, 보안사고 0건, 감사 적합 100%
- 권한: **L3**

---

## 평가·품질본부
- 입력: 변경분(PR/CR), 테스트 자산, 성능 목표
- 출력: 회귀/통합 테스트, 벤치마크, 품질 게이트 결과(통과/차단)
- KPI: 회귀 탐지율, 성능 회귀 0건, 변경 실패율 감소 기여
- 권한: **L1(차단권)**

---

## 개발자경험본부(DX)
- 입력: 플랫폼 표준, 사업부 요구, 운영/보안 가이드
- 출력: 표준 템플릿/SDK, 내부 도구, 문서/튜토리얼/ADR
- KPI: 온보딩 시간 단축, 템플릿 재사용률, 문서 최신화율
- 권한: L0~L1

---

## 고객성공본부
- 입력: 고객 요청/환경, 운영 인시던트, 제품 가이드
- 출력: 온보딩 플랜, 해결 가이드, 공지/상태 커뮤니케이션
- KPI: 온보딩 완료율, 문의 해결 리드타임, 공지 리드타임
- 권한: 공지는 L2 경로(운영본부/비서실 승인)

---

## 경영지원본부
- 입력: 사용량/예산 요청, 벤더/계약 요청
- 출력: 예산/결산, 조달/계약, 자산/계정 관리, 비용 리포트
- KPI: 예산 준수율, 계약 리드타임, 단가 최적화
- 권한: CFO 라인 L4(예산), 운영은 L2~L3

---

## 인사본부
- 입력: 역할 수요, 성과 데이터, 고충 접수
- 출력: 충원/배치, 평가/승진, 고충 조정, 교육/온보딩
- KPI: 충원 리드타임, 고충 해결 리드타임, 교육 완료율
- 권한: CHRO 라인 **L4**

---

## 5대 원칙 적용

이 문서는 다음 원칙을 반영합니다:

| 원칙 | 적용 방법 |
|------|----------|
| P1 단일 산출물 책임 | 각 공통본부가 고유 산출물만 담당. 평가·품질본부=품질 판정, 보안본부=정책, 운영본부=배포 승인 |
| P2 실행-검토 분리 | 공통본부는 게이트키퍼 역할. 사업부가 만든 산출물을 독립적으로 검토/승인 |
| P5 정책=매니저 | 각 본부의 게이트 기준이 Policy-as-Code로 자동 강제. 본부장이 아닌 정책이 판정 |

### 검증 포인트
- [ ] 각 본부의 게이트가 정책 기반으로 자동 운영되는가?
- [ ] 게이트 통과/반려가 자동 강제되는가 (수동 예외 최소화)?
- [ ] 실행 조직과 검토 조직이 완전히 분리되어 있는가?
