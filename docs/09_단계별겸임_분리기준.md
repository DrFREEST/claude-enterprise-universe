---

## 2) 분리 촉발 신호
- 리뷰 대기 병목 → 품질 분리
- 배포 실패 반복 → 운영/릴리즈 분리
- 표준 충돌/중복 개발 → 플랫폼/DX 분리
- 정책 예외 요청 증가 → 보안/정책 분리
- 회귀/성능 사고 증가 → 평가·품질 분리
- 고충/충원 이슈 증가 → 인사/고충 분리

## 5대 원칙 적용

이 문서는 다음 원칙을 반영합니다:

| 원칙 | 적용 방법 |
|------|----------|
| P1 단일 산출물 책임 | 겸임→분리 전환의 핵심 기준이 산출물 충돌 여부. 동일 산출물 유형 내 겸임은 허용, 이종 산출물 겸임은 분리 대상 |
| P2 실행-검토 분리 | 겸임 상태에서도 Maker-Checker 분리는 절대 원칙. 같은 에이전트가 생성+검토를 겸하는 것은 어떤 단계에서도 불허 |

### 검증 포인트
- [ ] 겸임/분리 기준이 산출물 유형 충돌에 기반하는가?
- [ ] Garage→Crew→Division 전환 시 Maker-Checker 분리가 유지되는가?
- [ ] 분리 트리거(월간 50건 초과)가 명시되어 있는가?
