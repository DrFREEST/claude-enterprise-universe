## 5대 원칙 적용

이 문서는 다음 원칙을 반영합니다:

| 원칙 | 적용 방법 |
|------|----------|
| P2 실행-검토 분리 | RACI에서 R(실행)과 A(승인)를 동일 에이전트/팀에 배정하지 않음. A≠R 원칙 엄수 |
| P3 티켓 기반 실행 | RACI의 모든 R(Responsible) 항목은 티켓(Task)으로 추적. C/I도 해당 티켓에 태그 |

### 검증 포인트
- [ ] 모든 활동에서 R(실행)과 A(승인)가 분리되어 있는가?
- [ ] RACI 항목이 티켓 시스템과 연동되어 추적 가능한가?
- [ ] Maker-Checker 관계가 RACI 매핑에 반영되어 있는가?
