# 회의체와 거버넌스 운영(Enterprise 기준)
- 작성일: 2026-02-12

---

## 1) 회의체(위원회)
- 경영위원회(월 1회, 분기 확장)
- 변경관리위원회(CAB)(주 2~3회 + 긴급)
- 품질게이트 회의(주 1회)
- 보안·정책 심의(주 1회)
- 운영 리뷰(주 1회)

---

## 2) 운영 리듬
- 매일: 운영 트리아지(알림/장애)
- 주 2~3회: CAB
- 주 1회: 품질게이트/보안정책/운영리뷰
- 격주/주간: 릴리즈 열차
- 월 1회: 경영위원회
- 분기: OKR/로드맵 리프레시

---

## 3) 회의 산출물(필수)
- 의사결정 ID/날짜/참석
- 배경/문제정의/대안/선택
- 영향 범위/리스크
- 롤백/대응 계획
- 담당(R)/승인(A)

---

## 5대 원칙 적용

이 문서는 다음 원칙을 반영합니다:

| 원칙 | 적용 방법 |
|------|----------|
| P3 티켓 기반 실행 | 모든 회의 결정사항은 티켓(Task)으로 기록. 구두 합의 불가, 반드시 티켓 ID 참조 |
| P4 기억 계층 분리 | 경영위원회 결정=L3/L4, 본부장 회의=L2, 프로젝트 스탠드업=L1. 회의 수준별 의사결정 범위 분리 |

### 검증 포인트
- [ ] 모든 의사결정이 티켓으로 기록되는 프로세스가 있는가?
- [ ] 회의체 수준이 기억 계층(L0~L4)과 매핑되어 있는가?
- [ ] 상위 회의체 결정이 하위 회의체를 오버라이드하는 구조가 명확한가?
