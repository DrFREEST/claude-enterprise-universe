# 보안심사 패키지(증적) 템플릿 — 엔터프라이즈 고객용
- 버전: 1.0
- 작성일: 2026-02-12
- 목적: 엔터프라이즈 고객 보안심사 대응을 위해 “필수 증적”을 일관되게 제공

---

## 1) 개요
- 고객/프로젝트:
- 심사 범위(제품/테넌트/연동):
- 제출 일자:
- 담당 조직:
  - 엔터프라이즈 연동 사업부(심사대응팀):
  - 보안·정책·준법본부:
  - 운영본부:
  - 평가·품질본부:

---

## 2) 아키텍처/데이터 흐름
- 시스템 아키텍처 개요(텍스트/그림 링크)
- 데이터 흐름(수집→처리→저장→삭제)
- 데이터 분류(일반/민감/개인정보) 및 처리 위치

---

## 3) 인증/권한(SSO/SCIM/RBAC)
- SSO 방식(OIDC/SAML) 구성 요약
- SCIM 프로비저닝 방식 요약
- RBAC 역할 정의/권한 매핑 표
- 관리자 권한 통제(2인 승인 등)

---

## 4) 보안 통제(필수 항목)
- 비밀정보 관리(저장/회전/접근 통제)
- 네트워크 통제(아웃바운드 제한/허용 목록)
- 런타임 격리(샌드박스/권한 최소화)
- 취약점 관리(스캔 주기/패치 정책)
- 로그/감사(누가/무엇/왜 변경했는지)

---

## 5) 준법/개인정보
- 개인정보 처리 여부/범위
- 데이터 보존/삭제 정책
- 로그 보존 기간 및 접근 통제
- 사고 대응 프로세스(고객 통지 포함)

---

## 6) 운영/가용성(SLO)
- 가용성 목표(SLO) 및 지표 정의
- 장애 대응(Sev 기준/MTTR 목표)
- 변경관리(CAB) 프로세스 요약
- 릴리즈 방식(카나리/점진/롤백)

---

## 7) 품질/테스트 증적
- 회귀/통합 테스트 리포트(링크)
- 성능/부하 테스트 결과(링크)
- 변경 실패율/롤백율 추이(요약)

---

## 8) 정책/게이트 증적(Policy-as-Code)
- 적용 중인 정책 버전/요약
- 정책 위반 방지 메커니즘(예: 커밋 키 탐지)
- 예외 승인 절차 및 기록 샘플(필요 시)

---

## 9) 제출 체크리스트
- [ ] 아키텍처/데이터 흐름
- [ ] 인증/권한 문서(SSO/SCIM/RBAC)
- [ ] 보안 통제(비밀/네트워크/격리/취약점/감사로그)
- [ ] 준법/개인정보(보존/삭제/통지)
- [ ] 운영(SLO/인시던트/CAB)
- [ ] 품질/테스트(회귀/성능)
- [ ] 정책 버전/게이트 설명

---

## 10) 승인
- 보안·정책·준법본부 승인:
- 엔터프라이즈 연동 사업부 승인:
- 최종 제출 승인(비서실/홍보 경로 필요 시):

---

## 5대 원칙 적용

| 원칙 | 적용 방법 |
|------|----------|
| P3 티켓 기반 실행 | 보안심사 패키지 제출이 티켓으로 추적 |
| P2 실행-검토 분리 | 증적 작성자와 심사 판정자가 분리 |
| P5 정책=매니저 | 보안 정책이 심사 기준으로 자동 강제. 정책 위반 시 심사 자동 반려 |
