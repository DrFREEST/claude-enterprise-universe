# 에스컬레이션 매트릭스 — 의사결정 교착/분쟁 해소 경로
- 버전: 1.0
- 작성일: 2026-02-12
- 목적: 조직 간 의사결정 교착 또는 분쟁 시 상승(escalation) 경로 정의
- 원칙: 각 레벨별 24시간 내 해결 의무 (원칙 3: 티켓 기반)

---

## 1) 에스컬레이션 레벨 구조

```
[Level 4] CEO 최종 결정 (48시간 내)
    ↑
[Level 3] C-level 판정 (24시간 내)
    ↑
[Level 2] 본부장 조율 (24시간 내)
    ↑
[Level 1] 팀 내 조율 (24시간 내)
    ↑
[Level 0] 에이전트 간 자율 해결 (즉시)
```

---

## 2) Level 0: 에이전트 간 자율 해결

### 대상 상황
- 단일 티켓 내 에이전트 간 기술적 의견 차이
- 코드 리뷰 피드백 반영 방법 논의
- 테스트 커버리지 목표 조율

### 해결 방법
- 에이전트 간 SendMessage로 조율
- RACI 매트릭스 참조 (원칙 2: 실행-검토 분리)
- 정책 문서(L3) 우선 참조 (원칙 4)

### 에스컬레이션 트리거
- 2시간 내 합의 실패
- 정책 해석 충돌
- 권한 범위 불명확

---

## 3) Level 1: 팀 내 조율 (24시간 이내)

### 대상 상황
- 팀 내 우선순위 충돌
- 리소스 할당 조율 필요
- 기술 선택 의견 불일치

### 해결 주체
- 팀 리더 (Tech Lead, QA Lead 등)
- 동일 BU 내 시니어 에이전트

### 해결 방법
1. 티켓에 `escalation: L1` 태그 추가 (원칙 3)
2. 팀 리더가 양측 의견 청취
3. 정책(L3) 및 프로젝트 컨텍스트(L1) 기반 판정
4. 판정 결과를 ADR(Architecture Decision Record)로 기록

### 시간 제한
- 에스컬레이션 접수 후 24시간 내 판정
- 판정 지연 시 자동으로 Level 2 상승

### 에스컬레이션 트리거 (Level 2로 상승)
- 24시간 내 판정 실패
- 정책 변경 필요
- 타 BU 영향 발생

---

## 4) Level 2: 본부장 조율 (24시간 이내)

### 대상 상황
- BU 간 의존 관계 충돌
- 정책 해석 차이
- 리소스 배분 분쟁 (예산, 인원, 인프라)

### 해결 주체
- 해당 본부장 (CTO, COO, CISO 등)
- 관련 BU 리더 참여

### 해결 방법
1. 티켓에 `escalation: L2` 태그 추가
2. 본부장이 BU 리더 소집 (가상 회의)
3. RACI 매트릭스 기준 책임 명확화 (원칙 1, 2)
4. 합의 또는 본부장 판정
5. 판정 결과 ADR 기록 + 관련 정책 업데이트 검토

### 시간 제한
- 에스컬레이션 접수 후 24시간 내 판정
- 판정 지연 시 자동으로 Level 3 상승

### 에스컬레이션 트리거 (Level 3로 상승)
- 24시간 내 판정 실패
- 본부 간 이해 충돌 (예: 보안 vs 운영)
- 전사 정책 변경 필요

---

## 5) Level 3: C-level 판정 (24시간 이내)

### 대상 상황
- 본부 간 근본적 이해 충돌
  - 보안 vs 운영 (CISO vs COO)
  - 품질 vs 속도 (CTO vs CPO)
  - 비용 vs 기능 (CFO vs CPO)
- CAB(변경승인위원회) 교착
- 전사 정책 변경 필요

### 해결 주체
- 관련 C-level 임원들
- 필요 시 CEO 중재

### 해결 방법
1. 티켓에 `escalation: L3` 태그 추가
2. 관련 C-level 임원 긴급 회의 소집
3. 각 본부 입장 정리 (데이터 기반)
4. 전사 우선순위(OKR) 기준 판정
5. 판정 결과 전사 공지 + 정책 업데이트

### 시간 제한
- 에스컬레이션 접수 후 24시간 내 판정
- 판정 지연 또는 C-level 간 합의 불가 시 Level 4 상승

### 주요 충돌 시나리오

#### 시나리오 1: 보안 vs 운영 (CISO vs COO)
- **상황**: 긴급 배포 vs 보안 게이트 통과 실패
- **판정 주체**: CISO + COO 공동 판정
- **합의 불가 시**: CEO 최종 판정 (Level 4)
- **판정 기준**: 리스크 vs 비즈니스 영향 정량화

#### 시나리오 2: 품질 vs 속도 (CTO vs CPO)
- **상황**: 기능 출시 일정 vs 테스트 커버리지 미달
- **판정 주체**: CTO + CPO 공동 판정
- **합의 불가 시**: CEO 최종 판정 (Level 4)
- **판정 기준**: 고객 영향 vs 기술 부채 정량화

#### 시나리오 3: 비용 vs 기능 (CFO vs CPO)
- **상황**: 예산 초과 기능 개발 vs 고객 약속
- **판정 주체**: CFO + CPO 공동 판정
- **합의 불가 시**: CEO 최종 판정 (Level 4)
- **판정 기준**: ROI vs 고객 이탈 리스크

### 에스컬레이션 트리거 (Level 4로 상승)
- C-level 간 합의 불가
- 전사 전략 방향 결정 필요
- 법적/윤리적 이슈 포함

---

## 6) Level 4: CEO 최종 결정 (48시간 이내)

### 대상 상황
- C-level 간 합의 불가
- 전사 전략 방향 결정
- 법적/윤리적/PR 리스크 포함
- 외부 이해관계자(고객, 규제기관) 관련

### 해결 주체
- CEO (최종 의사결정권자)
- 필요 시 이사회 자문

### 해결 방법
1. 티켓에 `escalation: L4` 태그 추가
2. CEO에게 종합 브리핑 (각 본부 입장 + 데이터)
3. CEO 최종 판정
4. 판정 결과 전사 공지
5. 관련 정책/전략 문서 업데이트
6. ADR로 기록 (향후 유사 상황 참조)

### 시간 제한
- 에스컬레이션 접수 후 48시간 내 판정
- CEO 판정은 최종 결정이며 재상신 불가 (단, 새로운 증거 발견 시 예외)

### 판정 기준
- 전사 OKR 및 미션 부합성
- 장기 전략 방향성
- 외부 영향 (고객, 시장, 규제)
- 조직 문화 및 원칙 준수

---

## 7) 특수 케이스: CAB 교착 해소

### CAB 구성 (원칙 2: 실행-검토 분리)
- 운영본부 (주관)
- 평가·품질본부
- 보안·정책·준법본부
- 해당 사업부 대표

### CAB 교착 시나리오
- 운영본부: 배포 승인
- 보안본부: 보안 게이트 차단
- → 합의 불가

### 에스컬레이션 경로
1. **1차 조율**: CAB 내 본부장급 조율 (Level 2)
2. **2차 조율**: COO + CISO 공동 판정 (Level 3)
3. **최종**: CEO 판정 (Level 4)

### 시간 제한
- CAB 회의 당일 1차 조율 완료 시도
- 24시간 내 2차 조율
- 48시간 내 CEO 최종 판정

## 9) 에스컬레이션 KPI 및 모니터링

|------|------|----------|
| Level 0 해결률 | >90% | 주간 |
| Level 1 24h 준수율 | >95% | 주간 |
| Level 2 24h 준수율 | >90% | 주간 |

### 월간 리포트 (COO 주관)

- 시간 준수율 (24h/48h)
- 개선 제안 (정책 명확화, RACI 조정 등)

## 10) 에스컬레이션 방지 정책 (원칙 5: 정책=매니저)

### 사전 예방
- 명확한 RACI 매트릭스 (원칙 1, 2)
- 정책 문서 정기 업데이트 (분기 1회)
- 크로스 BU 협업 SOP 준수
- 주요 의사결정은 ADR로 사전 기록

### 반복 에스컬레이션 방지
- 동일 이슈로 월 3회 이상 에스컬레이션 → 정책 즉시 업데이트
- 근본 원인 제거 (불명확 권한, 프로세스 갭 등)
- 예방 조치 문서화
