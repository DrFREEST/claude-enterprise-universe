# 사업부 운영 플레이북 (BU 5개, Enterprise 기준)
- 작성일: 2026-02-12
- 공통 원칙: 사업부는 L0~L1까지만 소유, 배포 L2/정책 L3는 중앙 최종

---

## 공통: 사업부 내부 최소 조직(권장)
- 제품기획팀(기획)
- 제품개발팀(개발)
- 도메인 QA팀(사업부 내부 검증)
- 운영협업 담당(운영본부/고객성공과 조율)
- (필요 시) 솔루션/현장 적용팀

---

## BU1 워크플로우 자동화 사업부
### 팀
- 제품기획팀 / 워크플로우 엔진팀 / 자동화 커넥터팀 / 도메인 QA팀 / 고객 솔루션팀
### 입력
- 고객 자동화 요구, 템플릿/SDK, 정책 규칙, 운영 지표(실패 케이스)
### 출력
- 워크플로우 기능, 커넥터, 템플릿, 오류 복구 로직, 사용자 가이드
### KPI
- 워크플로우 실행 성공률, 재시도 후 회복률
- 커넥터 연동 성공률, 오류 재발률
- 고객 TTV, 템플릿 채택률/재사용률
### 권한
- L0(구현/개선), L1(도메인 QA 승인)
- L2 배포는 운영본부 승인, L3 정책 변경은 보안본부 승인

---

## BU2 프로덕션 빌더 사업부
### 팀
- 제품기획팀 / 빌드 파이프라인팀 / 배포 전략팀 / 환경·구성관리팀 / 도메인 QA팀
### 입력
- 릴리즈 정책(운영본부), 보안 정책(보안본부), 플랫폼 런타임, 고객 배포 요구
### 출력
- 빌드/테스트/패키징 자동화, 배포 템플릿, 환경 표준, 롤백 레시피
### KPI
- 배포 리드타임, 빌드 성공률
- 변경 실패율/롤백율 감소 기여, 핫픽스 빈도 감소
- 표준 배포 템플릿 적용률
### 권한
- L0~L1(도구/템플릿 제공 및 자체 검증)
- 최종 배포권은 운영본부(L2), 정책은 보안본부(L3)

---

## BU3 엔터프라이즈 연동 사업부
### 팀
- 엔터프라이즈 제품기획팀 / 인증·권한 연동팀 / 레거시·데이터 연동팀 / 보안심사 대응팀 / 도메인 QA팀
### 입력
- 고객 보안요구서, 준법 체크리스트, SSO/SCIM 요구, 데이터 경계 요구
### 출력
- SSO/SCIM/RBAC 매핑, 레거시 연동 커넥터, 심사 증적 패키지
### KPI
- 보안심사 통과율/리드타임
- 연동 성공률, 장애 MTTR
- 계약→가동 리드타임
### 권한
- L0~L1(연동 개발/검증)
- 정책/준법 변경은 중앙(L3), 대외 문서는 비서실/홍보 경유(L2)

---

## BU4 AI 운영·거버넌스 사업부
### 팀
- 운영정책 제품기획팀 / 관제·알림 자동화팀 / 변경관리 자동화팀 / 감사·증적 자동화팀 / 운영 QA팀
### 입력
- 운영 인시던트 데이터, 감사 요구, 정책 엔진, 품질 게이트 결과
### 출력
- 런북 자동화, CAB 자동화 도구, 감사 리포트 자동화, 관제 대시보드
### KPI
- 탐지 리드타임, 알림 정확도(오탐/미탐)
- MTTR 감소 기여, 변경 승인 리드타임 감소 기여
- 증적 자동 생성 커버리지, 감사 대응 리드타임
### 권한
- L0~L1(도구 제공)
- 실제 운영 실행 L2는 운영본부, 정책 L3는 보안본부

---

## BU5 에이전트·템플릿 마켓 사업부
### 팀
- 마켓 제품기획팀 / 템플릿·레시피팀 / 검수·심사팀 / 개발자 생태계팀 / 마켓 QA팀
### 입력
- 플랫폼 API/SDK, 정책 규칙, 사용자 피드백, 파트너 요청
### 출력
- 템플릿/플러그인, 마켓 운영 규정, 검수 리포트, 생태계 가이드
### KPI
- 템플릿 채택률/재사용률, 활성 공급자 수
- 검수 리드타임, 정책 위반 반려율, 악성 등록 0건
- 문제 신고율/환불율 감소
### 권한
- 검수·심사팀은 L1(등록 승인/반려)
- 정책 규칙 편집은 중앙(L3), 약관/정산은 CFO 경유(L4)

## 5대 원칙 적용

이 문서는 다음 원칙을 반영합니다:

| 원칙 | 적용 방법 |
|------|----------|
| P1 단일 산출물 책임 | 각 사업부(BU)의 팀이 고유한 산출물만 담당. 기획팀=PRD, 개발팀=코드, QA팀=테스트 결과 |
| P3 티켓 기반 실행 | 모든 BU 작업은 티켓(TaskCreate)에서 시작. 구두 지시나 암묵적 합의 없이 추적 가능한 작업만 수행 |

- [ ] 모든 작업 프로세스가 티켓 생성에서 시작하는가?
- [ ] 팀 간 산출물 겸임이 발생하지 않는가?
